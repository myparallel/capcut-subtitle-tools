# CapCut Subtitle Tools — 剪映字幕工具集

**两个 opencode Skill，解决剪映视频创作中文案、字幕、配音对齐的痛点。**

---

## 痛点

做剪映视频时，最消耗精力的往往不是画面剪辑，而是这三件事：

1. **有文案，没配音** — 写好稿子了，还得自己录、找人录，或者用 TTS 生成后再手动对齐时间轴
2. **有录音，没字幕** — 口播录好了，剪映自动识别的文字有错别字，逐句修改费时费力
3. **有字幕，对不齐** — 改一个字，音轨长度变了，配音又要重来；中英双语更麻烦，一个改了另一个没改，不同步

这些问题叠加在一起，让视频创作者在画面构思之外还要耗费大量精力在文案-字幕-音频的对齐和纠错上，严重影响了创作效率与乐趣。

---

## 方案

从小处着手，把最常见的两种输入场景做成标准流程：

### 场景一：有文案，没配音 → 文生音字幕器

**输入：** TXT 文案文件  
**输出：** 配音 MP3 + 上英下中双语 SRT

流程：
1. 分段 + 翻译成双语
2. 用 Edge TTS 自然语速朗读
3. 按实际朗读时长修正时间轴
4. 输出配音与字幕，时间轴完全对齐

### 场景二：有录音，没字幕 → 音生文字幕器

**输入：** 音频文件 + 文案稿（可选）  
**输出：** 上英下中双语 SRT

流程：
1. 用 Whisper STT 获取精确时间轴
2. **文案稿校准机制**：用无错字的基准文稿，逐句替换 STT 识别错误
3. 保持 STT 原始时间点不变，只替换文字
4. 输出与音频严格同步的双语字幕

---

## 统一标准

所有输出的字幕统一为 **上英下中** 格式：

```
1
00:00:00,000 --> 00:00:05,280
On this quiet night, our foreign trade salespersons are still working overtime in the office,
在这个宁静的夜晚，我们的外贸业务员仍在办公室加班，
```

剪映导入后直接使用，无需二次调整。

---

## 使用方式

这两个工具是 **opencode**（AI 编程助手）的 Skill，放入项目的 `.opencode/skills/` 目录后即可调用。

### 文生音字幕器

```
"帮我根据这个文案做字幕"
```

→ 自动生成配音 MP3 + 双语 SRT

### 音生文字幕器

```
"帮我根据这个音频做字幕，文案稿在这里"
```

→ 自动生成与音频精确同步的双语 SRT

## 依赖

| 工具 | 用途 | 安装 |
|---|---|---|
| Python 3.10+ | 运行脚本 | — |
| `edge-tts` | 语音合成（文生音） | `pip install edge-tts` |
| `faster-whisper` | 语音识别（音生文） | `pip install faster-whisper` |
| FFmpeg | 音频处理 | `winget install ffmpeg` |
