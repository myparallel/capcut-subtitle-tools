# 文生音字幕器

从一段视频文案（TXT 文本）到剪映可直接导入的 **SRT字幕** + **配音MP3**，字幕与音频时间轴完全对齐。

## 工作流总览

```
原始文案 (TXT)
    │
    ▼  步骤1
分段 + 翻译 → bilingual segments [{en, zh}, ...]
    │
    ▼  步骤2 (CLI)
create_bilingual_srt → 上英下中双语SRT ✓
    │
    ▼  步骤3
用户选择配音语种（英文/中文）
    │
    ▼  步骤4 (CLI)
tts_from_srt → 自然语速配音MP3 + 时间轴修正的SRT ✓
```

---

## 技能依赖

运行前先检查依赖是否就绪：

```powershell
# 检查 Python 包
pip list 2>$null | findstr edge-tts
if ($?) { echo "edge-tts OK" } else { pip install edge-tts 2>&1 | out-null }

# 检查 ffmpeg
ffmpeg -version 2>$null
if ($?) { echo "ffmpeg OK" } else { winget install ffmpeg 2>&1 | out-null }
```

技能自带的脚本位于 `scripts/` 目录下：
- `generate_bilingual_srt.py` — 从双语文本段生成 SRT
- `tts_from_srt.py` — 从双语 SRT 生成配音 + 修正时间轴

---

## 步骤1：理解文案 + 分段 + 翻译

### 1.1 找到文案文件

- 让用户提供 TXT 文件路径，或在工作目录中查找 `*.txt`
- 用 `Read` 工具读取文件内容

### 1.2 确定源语言

- 判断文案内容是英文还是中文
- 如果用户没有说明，询问用户文案是什么语言，是否需要翻译

### 1.3 分段

将文案按语义拆成适合字幕显示的短句，每条字幕遵循以下原则：
- **每条字幕可读时间 2~6 秒**
- **语义完整** — 不要在句子中间断开
- **英文不超过 15 个词，中文不超过 20 个字**（可酌情放宽）
- 保持段落之间的自然停顿

### 1.4 翻译

为每条字幕生成另一语言版本：
- 源语言为英文 → 翻译成中文
- 源语言为中文 → 翻译成英文

翻译要求：
- 译文自然口语化，适合朗读
- 不需要逐字对应，意思准确即可
- 专业术语保持一致性（如 IoT R&D → 物联网研发）

### 1.5 保存为 JSON

将分段和翻译结果保存为 `segments.json`，格式如下：

```json
[
  {"en": "On this quiet night...", "zh": "在这个宁静的夜晚..."},
  {"en": "having a video conference...", "zh": "正在与美国客户..."}
]
```

---

## 步骤2：生成初版双语 SRT

运行 `generate_bilingual_srt.py` 创建初始 SRT 文件（含估算时间轴）：

```powershell
python scripts/generate_bilingual_srt.py --segments segments.json --output 双语字幕_上英下中.srt
```

这会产生一个带估算时间轴的**上英下中**双语 SRT 文件，以备后续步骤使用。

生成的 SRT 格式：
```
1
00:00:00,000 --> 00:00:06,250
On this quiet night, our foreign trade salespersons are still working overtime in the office,
在这个宁静的夜晚，我们的外贸业务员仍在办公室加班，

2
00:00:06,549 --> 00:00:10,050
having a video conference with American clients.
正在与美国客户进行视频会议。
```

时间轴估算方式：英文 2.5 词/秒，中文 4 字/秒，每条至少 2 秒，条目间间隔 0.3 秒。

---

## 步骤3：选择配音语种

询问用户要用**英文**还是**中文**朗读。
- 如果用户之前已经明确说过，直接使用，不需要再问。

---

## 步骤4：生成配音 + 修正时间轴

运行 `tts_from_srt.py`：

```powershell
# 英文配音
$env:LANG_CHOICE="1"; python scripts/tts_from_srt.py --input 双语字幕_上英下中.srt

# 中文配音
$env:LANG_CHOICE="2"; python scripts/tts_from_srt.py --input 双语字幕_上英下中.srt
```

或者用参数形式过一遍流程也可以。此脚本会：
1. 读取双语 SRT
2. 用 Edge TTS 按选择语种逐条朗读（Microsoft 高质量神经网络语音）
3. 测量每条的真实朗读时长
4. 用实际时长重建时间轴
5. 输出：
   - `配音字幕_双语_配音英文.srt`（或中文）— 时间轴修正后的双语字幕
   - `配音_英文.mp3`（或中文）— 对齐的配音音频

**关键原则：** 音频以自然语速朗读，绝不做变速处理。字幕时间完全跟随朗读时长。

---

## 输出交付

完成后告知用户以下配对文件：

| 语种 | 字幕文件（上英下中） | 配音文件 |
|---|---|---|
| 英文 | `配音字幕_双语_配音英文.srt` | `配音_英文.mp3` |
| 中文 | `配音字幕_双语_配音中文.srt` | `配音_中文.mp3` |

剪映导入方法：
1. 导入视频素材到时间轴
2. 「音频」→「导入音频」→ 选择对应的 `.mp3`
3. 「文本」→「导入字幕」→ 选择对应的 `.srt`
4. 在轨道上选中字幕，右侧样式面板调整为上英下中双行显示

---

## 注意事项

- **不要对生成的音频做变速处理** — 这是本技能区别于其他方案的核心优势
- 如果用户没有 TXT 文件而是直接给文案，同样可以先保存为文件再走流程
- 如果用户已经有双语 SRT 但需要配音，可以直接从步骤3开始
- `tts_from_srt.py` 的输入 SRT 必须是**上英下中**格式（第1行英文、第2行中文），其他格式会解析失败
